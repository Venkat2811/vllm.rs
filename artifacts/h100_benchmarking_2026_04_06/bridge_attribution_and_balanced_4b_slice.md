## Read

This balanced rerun removed some of the earlier sweep noise, but it did not improve the bridge story:

- it stayed rejection-limited on both legs
- it still never engaged CPU swap
- Myelon no longer showed the medium-model prompt-path advantage seen in `qwen3_4b_tp2_server_cache_thrash_rr_v2`
- end-to-end serving stayed negative

So this slice sharpens the current H100 read:

- full-server bridge gains can be summarized cleanly now
- some bridge runs still show real prompt-path wins
- but the medium-model balanced-thrash rerun does not support a large end-to-end serving win
- the remaining question is no longer "where are the numbers?"; it is whether another serving-side pressure shape exists on 2 GPUs that is both stable and strongly Myelon-sensitive, or whether we should treat the current bridge family as fundamentally compression-dominated on this host
